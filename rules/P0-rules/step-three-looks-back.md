# 🔴 一步三回头纪律（P0 — 每次任务自然断点执行）
> 优先级: P0 | 加载: always | 来源: 生产实践验证

**三步不依赖智能，依赖惯性。在自然断点（任务开始/并行发散前/提交前）执行，不停下来想"要不要做"。**

## 三问（按顺序）

### ① 能并行吗？
**触发时机**: 任务开始+每新增一个独立子任务
**自问**: "这个和哪个不冲突？"
**执行**: 不冲突的就并行启动。单个subagent能干的事不用等。

### ② 上一步对吗？
**触发时机**: 任何工具返回后、进入下一步前
**自问**: "刚返回的结果和我预期的差在哪？"
**执行**: 快速扫一眼输出——有error/异常/空结果就停，修了再走。没异常就过。
**禁止**: "看起来没问题" —— 用输出内容判断，不用印象判断。

### ③ 现有接口能复用吗？
**触发时机**: 准备写新代码/新轮子前
**自问**: "这个功能有没有现成的？"
**执行**: 
  - 先搜 `scripts/core-engine/` 同名或近名轮子
  - 再看 cad_pipeline_fanout.py 已注册阶段
  - 都无 → 新建
**禁止**: 明知道有轮子还自己写。

## 三连触发位置（硬编码到自然断点）

```
任务接收 → ①能并行吗？
                 ↓
子任务执行 → 每步 → ②上一步对吗？
                          ↓
                准备写代码 → ③现有接口能复用吗？
                                   ↓
             提交/交付 → ①②③快速复查
```

## 不依赖智能
这三问不需要"想"——在断点处自然触发就行。做多了变习惯，习惯变本能。
不做 → 回归单线程+重复造轮子+向前冲不回头看 = 旧模式。
